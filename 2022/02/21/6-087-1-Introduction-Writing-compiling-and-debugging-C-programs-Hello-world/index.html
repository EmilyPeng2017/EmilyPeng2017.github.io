6.087-1-Introduction. Writing, compiling, and debugging C programs. Hello world.

Posted on 2022-02-21 Edited on 2022-12-26 In C language Views: Symbols count in article: 937 Reading time ≈ 1 mins.
Introductory lecture presentation on how to edit, compile, and debug C programs, and C programming fundamentals.

1. What is C?

1.1 widely used today

a. extends to newer system architectures
b. efficiency/performance
c. low-level access --> often means faster

1.2 what is C used for?

a. OSes, like Linux
b. microcontrollers: automobiles and airplanes
c. embedded processors: phones, portable electronics, etc
d. DSP processors: digital audio and TV systems

1.3 C lacks:

a. exceptions
b. range-checking
c. garbage collection
d. object-oriented programming
e. polymorphism

2. Writing C Programs

2.1 steps

  1. edit .c file directly
  2. gcc compiler to get .o file
    gcc -Wall infilename.c -o outfilename.o
    // embed debugging info and disable optimization
    gcc -g -o0 -Wall infilename.c -o outfilename.o

3. Our First C Program

3.1 structure of a .c file

  1. begin with comments about file contents
  2. insert #include statements and preprocessor definitions
  3. function prototypes and variable declarations
  4. main() function
  5. define other functions

3.2 conditional preprocessor macros

#if, #ifdef, #ifndef, #else, #elif, #endif
